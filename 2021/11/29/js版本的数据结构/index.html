JavaScript 版本的数据结构

前言

老早就想找个机会把数据结构安排了!
这次就一次性安排上吧,可能篇幅会很长,慢慢完成吧.

一天学一点,不要小看时间的威力!

这其实算是一个学习笔记,与之配套的课程链接: JavaScript 版数据结构与算法 轻松解决前端算法面试

队列

队列的特点是先进先出,就好比你去学校食堂打饭排的队,先排队就能先打到饭.
js里并没有队列,但是可以使用array来模拟一个

如上图所示,我们使用数组成功模拟了一个队列.

栈

栈和队列是很相似的,只是它的特点是先出后出,在生活中的场景可以比作弹夹上弹,你后压进去的弹药会被先射出来.js里没有栈这个数据接口,不过我们依然可以使用array来模拟一个

栈这种数据其实与我们息息相关,比如函数调用堆栈call Stack,还有代码括号的有效性.

可以看到,和队列的区别也就一个先进后出的区别.

集合

集合是无序的,并且里边的元素是不可重复的,js里已经支持了Set这种数据结构.我们可以利用集合的不可重复性来进行数组去重等操作.

我们还可以使用Set来求两个数组的交集等等.

字典

字典是一种键值对集合的数据结构.你或许会问,我们平时使用的object不也是这样的数据结构吗,它有何好处?答案是他的键可以是any类型!字典的键是不可重复的,值是可以重复的.

js里已经支持了这种数据类型,我们来使用一下吧.

链表

链表和数组,队列,栈的最大区别就是,他在内存里并不是一整块空间,就是说不是连续的内存空间,它是通过引用指针来将一个一个分散的数据连接起来,形成链状数据;由于链表的这种数据结构,带来的优点是节点的增删速度是极快的

js里没有链表这种数据结构,不过我们可以使用对象来模拟链表.

这样,我们就基本实现了一个链表结构.可以看到,链表结构的数据增删是非常快的!只需要简单几步,不用循环遍历整个数据块.但是查找等操作将会非常耗时.

在链表里,指针是一个重要的概念,你可以在一些算法里看到指针的运用,双指针等等.

一个便于理解指针的例子,我们可以通过键数组来获取一个 json节点的值

树

树形数据结构,就像他的名字一样,整个是以树形结构来储存数据.当然其中也包含很多分支:比如经常听到的红黑树,二叉树;

js里没有树这个数据结构，但是我们可以通过array+object来模拟树结构的数据.

树的每个节点大致是下边这种结构.

叶子节点:集节点下边没有分支,是一个末尾节点,这样就被称之为叶子节点.

图

图这个数据结构,是由很多边链接形成的,并不是我们日常见到的图片的图.它是一个数据抽象模型,可以表示任何二元关系.
js里并没有图这个数据结构,但是我们可以使用array和object来进行模拟.

图的表示法一:邻接矩阵

邻接矩阵主要使用0和1来表示两个点直接的链接,如果两点之间有链接,那么就用1来表示,如果没有链接,那就用0来表示.
