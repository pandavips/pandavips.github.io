hooks的一些常识:

  • hooks只能在函数组件中使用,不能在类组件中使用
  • hooks只能在函数组件的顶层调用,不能在循环、条件或嵌套函数中调用(这是由于react通过调用hooks和组件渲染函数的调用时序来和组件进行关联的.)

只记录常用的hooks,不常用的hooks可以参考react官网-hooks进行学习.

useState(initialState) ​

useState 是一个 React Hook，它允许你向组件添加一个 状态变量。

用法 ​

jsx
const [index, setIndex] = useState(0);

参数 ​

  • 传入一个值类型,会把传入的值类型作为初始值
  • 传入一个同步函数类型,会惰性将函数的返回值作为初始值;如果这个时候你就是希望,它的状态是一个函数,那么你需要将函数进行包裹,就是传入一个返回该函数的高阶函数
jsx
// 如果传入的是函数
const [index, setIndex] = useState(()=>0);

// 如果期望将函数作为初始值
const [index, setIndex] = useState(() => () => 0);

特性/注意点 ​

React 16.8 & 17 示例：setState 不自动批处理 ​

在18之前,如果在异步之后进行多次状态更新,那么不出进行批处理更新,下边的例子中将会渲染两次

jsx
import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setTimeout(() => {
      setCount(count + 1); // 第一次 setState
      setCount(count + 2); // 第二次 setState
    }, 0);
  };

  console.log('Render:', count);

  return <button onClick={handleClick}>Count: {count}</button>;
}

React 18 示例：自动批处理 ​

在18以后的版本中,在异步中多次更新状态将会进行批处理,例如下边的例子中只会渲染一次

jsx
import React, { useState } from 'react';
import { createRoot } from 'react-dom/client';

function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    setTimeout(() => {
      setCount(c => c + 1);
      setCount(c => c + 2);
    }, 0);
  };

  console.log('Render:', count);

  return <button onClick={handleClick}>Count: {count}</button>;
}

const root = createRoot(document.getElementById('root'));
root.render(<Counter />);

useReducer(reducer, initialArg, init?) ​

useReducer 是一个 React Hook，它允许你向组件里面添加一个 reducer。

用法 ​

其实你就当做对setState的一个封装就行了.

jsx
import { useReducer } from "react";
const reducer = (state, action) => {
  switch (action.type) {
    case "ADD_TODO":
      return {
        ...state,
        todo: [
          ...state.todo,
          {
            id: action.payload.id,
            text: action.payload.text,
          },
        ],
      };
    case "ADD_DONE":
      return {
        ...state,
        done: [
          ...state.done,
          {
            id: action.payload.id,
            text: action.payload.text,
          },
        ],
      };
    // 任何其他操作...
    default:
      return state;
  }
};
export function UseReducerDemoView() {
  const [state, dispatch] = useReducer(reducer, {
    // 待做
    todo: [],
    // 已做
    done: [],
  });

  const addTodo = () => {
    dispatch({
      type: "ADD_TODO",
      payload: { id: Date.now(), text: "New Todo Item" },
    });
  };

  const addDone = () => {
    dispatch({
      type: "ADD_DONE",
      payload: { id: Date.now(), text: "New Done Item" },
    });
  };

  return (
    <div>
      <p>state:{JSON.stringify(state)}</p>
      <button onClick={addTodo}>Add Todo</button>
      <button onClick={addDone}>Add Done</button>
    </div>
  );
}

参数 ​

  • reducer：用于更新 state 的纯函数。参数为 state 和 action，返回值是更新后的 state。state 与 action 可以是任意合法值。
  • initialArg：用于初始化 state 的任意值。初始值的计算逻辑取决于接下来的 init 参数。
  • 可选参数 init：用于计算初始值的函数。如果存在，使用 init(initialArg) 的执行结果作为初始值，否则使用 initialArg。

特性/注意点 ​

其实就是在状态操作稍微复杂时,useState会显得力不从心,这个时候可以使用这个hooks将相关的逻辑封装到一起.否则单凭肉眼看,有时候一时半会是无法把多个state以及操作函数联系起来的.

useEffect(setup, dependencies?) ​

useEffect 是一个 React Hook，它允许你 将组件与外部系统同步。

用法 ​

jsx
import { useState, useEffect } from 'react';
import { createConnection } from './chat.js';

function ChatRoom({ roomId }) {
  const [serverUrl, setServerUrl] = useState('https://localhost:1234');
  useEffect(() => {
    const connection = createConnection(serverUrl, roomId);
    connection.connect();
    return () => {
      connection.disconnect();
    };
  }, [serverUrl, roomId]);
  // ...
}

它会返回一个函数,可以用于副作用的清理,它的执行时机:

  1. 组件卸载时（unmount）:当组件被卸载时，React 会自动调用 useEffect 返回的清理函数。
  2. 依赖项变化前（before effect re-run）:
  • 如果 useEffect 有依赖项，当这些依赖项变化时，React 会：先调用上一次的清理函数（如果有）
  • 然后执行新的 effect 回调

图示流程（React 行为）:

text
Mount:
- run effect
Update (if deps changed):
- cleanup previous
- run new effect
Unmount:
- cleanup

参数 ​

  • setup: 副作用函数，可选择性返回清理函数。组件挂载时执行setup，依赖项变化时先执行上次的清理函数再执行新的setup，组件卸载时执行最后的清理函数。

  • dependencies: 可选，包含effect中使用的所有响应式值（props、state、组件内变量/函数）的数组。必须使用固定长度并以[dep1, dep2, ...]形式内联编写。React使用Object.is比较依赖项。省略时每次渲染后都会执行effect。

我们快速来排查一下第二个参数不同的情况:

  • 传入空数组[]和不传:这两种参数的行为是等价的,都是只在组件挂载时执行一次.
  • 传入数组依赖项,如果数组中任意一项

特性/注意点 ​

这个hooks可以说是,非常灵活且强大，但是即使很有经验的react开发者也不敢说完全掌控了它.

useLayoutEffect(setup, dependencies?) ​

和useEffect类似,但是它会阻塞浏览器更新,所以我们一般使用useEffect就可以了,除非你需要在DOM更新之前执行一些操作,例如获取DOM的宽高等.

同步和异步执行要取决于你的场景,并没有绝对的好坏.

useRef(initialValue) ​

useRef 是一个 React Hook，它能帮助引用一个不需要渲染的值。

用法 ​

jsx
import { useRef } from 'react';
function Stopwatch() {
  const intervalRef = useRef(0);
  // ...
}

useRef 返回一个具有单个current属性 的 ref 对象，并初始化为你提供的 初始值。

参数 ​

initialValue：ref 对象的 current 属性的初始值。可以是任意类型的值。这个参数在首次渲染后被忽略。

特性/注意点 ​

current属性也可以用来保存状态,但是改变 ref 不会触发重新渲染.

使用 ref 可以确保：

  • 可以在重新渲染之间 存储信息（普通对象存储的值每次渲染都会重置）。
  • 改变它 不会触发重新渲染（状态变量会触发重新渲染）。
  • 对于组件的每个副本而言，这些信息都是本地的（外部变量则是共享的）。

useImperativeHandle(ref, createHandle, dependencies?) ​

主要和useRef配合使用,用于向父组件暴露自己定义的对象,除了current属性,还可以暴露其他的属性和方法.

useImperativeHandle 是一个 React Hook，它能让你自定义通过 ref 暴露给父组件的实例值。

用法 ​

jsx
import { forwardRef, useImperativeHandle, useRef } from 'react';

// 子组件使用 forwardRef 接收父组件传递的 ref
const FancyInput = forwardRef(function FancyInput(props, ref) {
  const inputRef = useRef(null);
  
  // 自定义暴露给父组件的实例值
  useImperativeHandle(ref, () => {
    return {
      // 只暴露我们想要父组件访问的方法
      focus: () => {
        inputRef.current.focus();
      },
      // 自定义方法
      setCustomValue: (value) => {
        inputRef.current.value = value.toUpperCase();
      },
      // 获取当前值
      getValue: () => inputRef.current.value
    };
  }, []);
  
  return <input ref={inputRef} {...props} />;
});

// 父组件
function Form() {
  const fancyInputRef = useRef(null);
  
  function handleClick() {
    // 可以调用我们暴露的方法
    fancyInputRef.current.focus();
    fancyInputRef.current.setCustomValue('hello world');
    console.log(fancyInputRef.current.getValue());
  }
  
  return (
    <div>
      <FancyInput ref={fancyInputRef} />
      <button onClick={handleClick}>
        操作输入框
      </button>
    </div>
  );
}

参数 ​

  • ref: 从 forwardRef 渲染函数中获得的第二个参数。
  • createHandle: 函数，返回你想要暴露的 ref 对象。React 会将其返回值提供给父组件的 ref。
  • dependencies: 依赖项数组。如果数组中的任何值发生变化，useImperativeHandle 会重新执行创建函数。可以省略此参数，此时每次重新渲染都会重新创建 handle。

特性/注意点 ​

  • 使用 useImperativeHandle 可以让你在使用 ref 时暴露自定义的实例值，而不是整个 DOM 元素或组件实例。
  • 它必须与 forwardRef 一起使用。
  • 这种模式有助于限制子组件的暴露，只暴露必要的功能，保持封装性。
  • 当你需要让父组件能够直接调用子组件特定方法时特别有用，例如表单控件的聚焦、视频播放控制等。

useContext(SomeContext) ​

react和vue一样,也是单向数据流.如果我们没有特殊手段,如果我想向我的孙子组件传递数据,那么我们就需要一层一层的向下传递,如果你的组件层级很深,那么你每层都需要传递一次,这就很麻烦了. 所以我们可以使用useContext来解决这个问题,比如我们有这样一个场景,切换主题.

提供组件:

jsx
import { createContext, useContext, useState } from 'react';
export const ThemeContext = createContext();
export function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  const toggleTheme = () => {
    setTheme(theme === 'light' ? 'dark' : 'light');
  };
  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}
export function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return context;
}

通畅在顶层组件执行注入:

jsx
import { ThemeProvider } from './context/ThemeContext';
import { Header } from './components/Header';
import { Content } from './components/Content';

function App() {
  return (
    <ThemeProvider>
      <div style={{ minHeight: '100vh' }}>
      <Header />
      <Content />
      </div>
    </ThemeProvider>
  );
}
export default App;

消费组件:

jsx
import { useTheme } from "../context/ThemeContext";
export function Header() {
  const { theme, toggleTheme } = useTheme();
  return (
    <header
      style={{
        padding: "1rem",
        backgroundColor: theme === "light" ? "#ffffff" : "#333333",
        color: theme === "light" ? "#333333" : "#ffffff",
        borderBottom: "1px solid #ddd",
      }}>
      <h1>主题切换示例</h1>
      <button onClick={toggleTheme}>切换到{theme === "light" ? "深色" : "浅色"}主题</button>
    </header>
  );
}

之后在项目中任何地方,你都可以通过上边的方式获取主题相关的值了.

memo + useMemo + useCallback ​

这三兄弟都是用来做性能优化的,均使用缓存的机制来实现优化,

useCallback(fn, dependencies) ​

这个hooks可以让你的函数创建不那么频繁,即使组件被重新渲染,也不会创建新的函数,而是返回之前的函数引用.

useMemo(calculateValue, dependencies) ​

在组件的顶层调用 useMemo 来缓存每次重新渲染都需要计算的结果,有点vue计算属性的意思,如果依赖项没有变化,那么就会返回之前的值,否则就会重新计算.

memo(Component, arePropsEqual?) ​

使用 memo 将组件包装起来，以获得该组件的一个 记忆化 版本。通常情况下，只要该组件的 props 没有改变，这个记忆化版本就不会在其父组件重新渲染时重新渲染。但 React 仍可能会重新渲染它：记忆化是一种性能优化，而非保证。

和useCallback,useMemo不同,memo使用浅比较,而非Object.is,当然你也可以使用自定义函数.

jsx
import { memo, useState } from 'react';

export default function MyApp() {
  const [name, setName] = useState('');
  const [address, setAddress] = useState('');
  return (
    <>
      <label>
        Name{': '}
        <input value={name} onChange={e => setName(e.target.value)} />
      </label>
      <label>
        Address{': '}
        <input value={address} onChange={e => setAddress(e.target.value)} />
      </label>
      <Greeting name={name} />
    </>
  );
}

const Greeting = memo(function Greeting({ name }) {
  console.log("Greeting was rendered at", new Date().toLocaleTimeString());
  return <h3>Hello{name && ', '}{name}!</h3>;
});

在此示例中，请注意 Greeting 组件在 name 更改时重新渲染（因为那是它的 props 之一），但是在 address 更改时不会重新渲染（因为它不作为 props 传递给 Greeting）.

memo使用自定义函数的示例:

jsx
import React from "react";

// 子组件
const UserCard = ({ user }) => {
  console.log("👤 UserCard rendered");
  return (
    <div>
      <h3>{user.name}</h3>
      <p>ID: {user.id}</p>
    </div>
  );
};

// 自定义比较函数，只比较 user.id
const areEqual = (prevProps, nextProps) => {
  return prevProps.user.id === nextProps.user.id;
};

// 使用 React.memo 包装组件并传入比较函数
const MemoizedUserCard = React.memo(UserCard, areEqual);

export default function App() {
  const [name, setName] = React.useState("Alice");
  const [id, setId] = React.useState(1);

  const user = { id, name };

  return (
    <div>
      <MemoizedUserCard user={user} />

      <button onClick={() => setName(name === "Alice" ? "Bob" : "Alice")}>
        Toggle Name
      </button>

      <button onClick={() => setId(id + 1)}>
        Increment ID
      </button>
    </div>
  );
}
