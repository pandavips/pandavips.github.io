NestJS CLI常用命令整理 ​

快速Nest CLI常用命令

以下是 NestJS CLI 的常用命令整理，帮助你快速上手开发和项目管理：

1. 项目创建与初始化 ​

  • 创建新项目
    bash
    nest new project-name
    初始化一个全新的 NestJS 项目，并自动安装依赖。

2. 核心生成器命令 ​

  • 生成模块 (Module)

    bash
    nest generate module module-name
# 简写
nest g mo module-name

  • 生成控制器 (Controller)

    bash
    nest generate controller controller-name
# 简写
nest g co controller-name

  • 生成服务 (Service)

    bash
    nest generate service service-name
# 简写
nest g s service-name

  • 生成资源 (Resource)
    一键生成模块、控制器、服务及实体（支持 REST 或 GraphQL）：

    bash
    nest generate resource resource-name
# 简写
nest g res resource-name

3. 其他常用生成器 ​

  • 生成中间件 (Middleware)

    bash
    nest generate middleware middleware-name
# 简写
nest g mi middleware-name

  • 生成过滤器 (Exception Filter)

    bash
    nest generate filter filter-name
# 简写
nest g f filter-name

  • 生成拦截器 (Interceptor)

    bash
    nest generate interceptor interceptor-name
# 简写
nest g in interceptor-name

  • 生成守卫 (Guard)

    bash
    nest generate guard guard-name
# 简写
nest g gu guard-name

  • 生成管道 (Pipe)

    bash
    nest generate pipe pipe-name
# 简写
nest g pi pipe-name

4. 运行与构建 ​

  • 启动开发模式（热更新）

    bash
    nest start --watch

  • 构建生产版本

    bash
    nest build

5. 测试相关 ​

  • 运行单元测试

    bash
    nest test
# 或使用 Jest 命令
npm run test

  • 生成测试文件

    bash
    nest generate service service-name --no-spec  # 不生成测试文件
nest generate service service-name            # 生成测试文件（默认）

6. 项目配置与信息 ​

  • 查看 Nest CLI 版本

    bash
    nest --version

  • 查看项目信息

    bash
    nest info

    显示当前项目依赖的 NestJS 版本、平台（Express/Fastify）等信息。

  • 更新 Nest CLI

    bash
    nest update

    更新 CLI 和项目依赖。

7. 其他实用命令 ​

  • 设置全局路径前缀
    在 main.ts 中设置：

    typescript
    app.setGlobalPrefix('api');

  • 生成自定义配置文件

    bash
    nest generate options
