prisma常用命令整理

以下是 Prisma 常用命令的整理，按功能分类：

1. 初始化与配置 ​

  • 初始化 Prisma 项目
    bash
    npx prisma init
    • 创建 prisma/schema.prisma 和 .env 文件。

2. 数据库迁移（Migrations） ​

  • 创建新迁移

    bash
    npx prisma migrate dev --name <迁移名称>
    • 生成迁移文件并应用更改到数据库。

  • 应用未执行的迁移（生产环境）

    bash
    npx prisma migrate deploy

  • 重置数据库并应用所有迁移

    bash
    npx prisma migrate reset

3. 数据模型同步 ​

  • 将 Schema 模型同步到数据库（不迁移）

    bash
    npx prisma db push

  • 从数据库拉取结构生成 Schema

    bash
    npx prisma db pull

4. 客户端生成 ​

  • 生成 Prisma Client
    bash
    npx prisma generate
    • 根据 schema.prisma 生成/更新客户端代码。

5. 数据管理 ​

  • 启动 Prisma Studio（图形化数据管理）
    bash
    npx prisma studio
    • 浏览器打开 http://localhost:5555 管理数据。

6. 调试与维护 ​

  • 格式化 Schema 文件

    bash
    npx prisma format

  • 检查 Schema 语法

    bash
    npx prisma validate

  • 查看 Prisma 版本

    bash
    npx prisma -v

  • 生成 ER 图（需第三方工具）

    bash
    npx prisma generate --schema=./prisma/schema.prisma
npx prisma-er-generator

7. 其他工具 ​

  • 升级 Prisma 相关依赖

    bash
    npx prisma-upgrade

  • 生成 Prisma 类型（TypeScript）

    bash
    npx prisma generate --data-proxy

常用组合场景 ​

  1. 开发环境修改模型后

    bash
    npx prisma migrate dev --name add_user_table
npx prisma generate

  2. 生产环境部署

    bash
    npx prisma migrate deploy
npx prisma generate

  3. 快速原型开发

    bash
    npx prisma db push  # 强制同步模型到数据库（慎用于生产）
npx prisma studio   # 手动添加测试数据

注意事项 ​

  • 确保 .env 中配置正确的数据库连接。
  • migrate dev 适用于开发环境，migrate deploy 用于生产。
  • 修改 Schema 后需重新执行 prisma generate 更新客户端。

建议结合 Prisma 官方文档使用：https://www.prisma.io/docs/
